# Lessons Learned Log

## Purpose

This file records mistakes, surprises, and durable lessons so the repository does not repeat avoidable confusion or wasted work.

## Lesson Ledger

| Date | Lesson | What Triggered It | Preventive Action |
|------|--------|-------------------|-------------------|
| 2026-03 | Chat history alone is not durable memory | Important decisions and rationale were at risk of remaining only in conversation context | Record accepted decisions and work outcomes in repo files and index them in `docs/work-index.md` |
| 2026-03 | A strong long-term ecosystem vision can still create short-term overdesign | Multi-system thinking repeatedly pulled planning away from a bounded first product | Keep current work centered on AI Architect MVP and require explicit entry criteria before opening the next specialist repo |
| 2026-03 | Review artifacts are useful, but they do not replace a compact operational ledger | Decisions were spread across review docs, memos, and conversation summaries | Maintain compact decision, discovery, lesson, and work logs alongside longer narrative documents |
| 2026-03 | For future sibling systems, separate fast spike work from ontology-forming research work | Architecture experiments and durable doctrine serve different purposes and should not be merged mentally or operationally too early | Use a spike branch to test shape and a research branch to codify validated patterns into AAA's durable ontology |
| 2026-03 | Static data-backed Next.js routes can serve stale cycle records if generated artifacts change after the last production build | The new ExMorbus cycle-history ledger was regenerated correctly, but `/cycles` still rendered old values until the frontend was rebuilt and restarted | After regenerating snapshot or cycle-history artifacts, rerun the frontend production build before treating browser validation as authoritative |
| 2026-04 | yt-dlp subtitle download fails silently when a comma-separated `--sub-lang` value is combined with `-f sb3`; the format selector parses the comma as part of the format string | All 20 YouTube transcript fetches returned no subtitle files until the bug was caught; the fix (`--sub-lang en` only, no comma) immediately resolved it | Always validate yt-dlp subtitle output with `ls tmpdir/*.json3 *.vtt` before trusting a silent success exit code |
| 2026-04 | Dedup checks that write to a persistent store on first ingest will silently skip re-enrichment unless the check is enrichment-aware | 92 LinkedIn posts were added text-only in the first pass, then permanently skipped by subsequent enrichment runs because `research_sources.json` already had their IDs | Design ingest dedup at field-level quality (e.g., skip only if `mentioned_tools` is non-empty), not just at ID presence |
| 2026-04 | Python's built-in hash() is non-deterministic across processes — never use it to generate persistent IDs | Two sequential runs of the blog ingest script produced different IDs for the same URLs, creating 120 duplicate entries in ChromaDB | Always use hashlib.md5 or sha1 for stable, reproducible IDs in any ingest pipeline |
| 2026-04 | A Chrome extension content script injected via popup requires `requireVersion` gating or the popup may run stale cached script from the prior session | Earlier extension versions sometimes executed against stale DOM state after a manifest or content-script update | Always increment `AAA_CONTENT_VERSION` in both `content.js` and `popup.js` on any scraping-logic change, and gate execution with a version check |
| 2026-06-28 | Important findings survived only in conversation context, not in the repo — when the session ended, the learning was gone | After multiple build sessions (ExMorbus, MCP server, ChromaDB, eval harness), the question "has AAA been effectively learning from these projects?" revealed that all three project logs existed but none were searchable; the work was recorded in files but the files weren't wired into the retrieval system; the gap was invisible until explicitly diagnosed | Always wire new log files into ChromaDB ingest on the same session they are created; "durable doc exists" is not the same as "system can find it" |
| 2026-06-28 | A 150-line CLAUDE.md with a priority-ranked Architectural Constitution outperforms a 500-line rule manifest | Three earlier versions of CLAUDE.md for AAA grew to 300+ lines; they covered known cases well but produced inconsistent behavior on novel situations because there was no conflict resolution mechanism; the Architectural Constitution (8 values, ranked) gives the model a way to derive correct behavior for cases no rule anticipated | Keep root CLAUDE.md under 150 lines; encode values and priority, not rules and checklists; depth goes in companion docs wired to the DoD |
| 2026-06-28 | Never mix internal institutional memory into the external knowledge corpus without a source tier | The first project-learning ingest dumped AAA's own decisions/lessons into the same ChromaDB collection as external thought leaders with no distinction; this polluted `get_trending_tools` with self-references (ExMorbus ×11) and let `get_architecture_recommendation` cite AAA itself as an external "thought leader" — a self-reinforcing echo chamber | Tag provenance with a `source_tier` (internal vs external) at ingest; exclude internal tier from trend/community aggregation; label internal priors distinctly in synthesis so they are treated as institutional memory, never independent authority |
| 2026-06-29 | The research_sources.json dedup bug RECURRED in a second ingest script — fixed structurally this time | The 2026-04 lesson (dedup against a historical superset permanently blocks re-ingest into a rebuilt store) reappeared in `process_linkedin_export.py`: it skipped on `load_existing_ids() \| store_ids`, so 58 of 62 store-new posts were silently dropped. Recurrence happened because the earlier fix was applied to one pipeline, not made a shared rule | Dedup the SKIP decision against the STORE (the authoritative target) in EVERY ingest script; use side-artifact logs (research_sources.json) only for their own append-dedup, never as the skip gate. Consider a shared `store.get_existing_ids()`-based dedup helper so this can't recur a third time |
| 2026-06-29 | When matching people by name token, verify identity — two different people shared a first name | Token-matching the target "Aishwarya Srinivasan" against the export hit "Aishwarya Naresh Reganti" (a different person), briefly overstating coverage | Confirm author identity (full name / profile URL), not a substring, before claiming a target persona is covered |
| 2026-06-28 | Enforce trust boundaries at the lowest shared data layer, not at one caller | Twice now a tier rule was enforced too high: source_tier trend exclusion lived in the MCP layer (self-references polluted trends), and experimental quarantine lived only in the MCP `search_knowledge` wrapper (direct `store.search()` callers — grounding, eval — leaked unpromoted artifacts). Each time, a second caller bypassed the rule | Put trust/quarantine/tier filters in the store/data layer so every caller inherits them by default; treat per-caller filtering as a smell. Prefer post-filtering over ChromaDB `$ne` (which silently drops docs missing the field) |
| 2026-06-28 | "Searchable internal memory" is not "learning from outcomes" — do not conflate retrieval with a feedback loop | The project-learning ingest was committed as "closes the feedback loop," but it only makes human-written notes searchable; the actual learning loop (recommendation → outcome capture → adjusted future behavior) was never built; the overstatement obscured that the thing the user actually asked about was still missing | Reserve "learning loop" / "feedback loop" for systems that capture outcomes and change behavior; describe retrieval honestly as retrieval; an adversarial review of one's own commit messages catches this category error |
