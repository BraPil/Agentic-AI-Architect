A candidate was asked this question in an LLM System Design Interview: "How would you reduce inference costs at scale?"
The answer always comes back to the same bottleneck, KV Cache and most people can't explain it clearly.

When an LLM generates a response, it computes attention over every prior token.

To avoid redoing that work on every step, it stores Key and Value vectors for each token in GPU memory.
That's the KV Cache.

The cache includes everything in context: system prompt, conversation history, your input, and output tokens generated so far.

A 128K context holds 32x more K/V pairs than a 4K one, 32x more GPU memory. Providers charge for that.

It's not just cost it's also speed. The initial prefill reads your full prompt once.
But every generated token attends over the full cache.
Longer context = slower generation, not just a bigger bill.

What to do:
→ Trim conversation history aggressively
→ Use retrieval instead of stuffing full docs into the prompt
→ Match your context window to what the task actually needs

The model isn't getting smarter with more context. It's getting more expensive.
