CEO of Y Combinator shared his 𝗖𝗟𝗔𝗨𝗗𝗘.𝗺𝗱 prompt for plan mode 🔥

It lets him ship 4,000+ line features with full tests in ~1 hour:

Garry doesn't ask Claude to write code first. He asks it to review the plan - like a staff engineer would before any changes are made.

The prompt forces Claude through a 4-part structured review:
1. Architecture - system design, dependencies, scaling risks
2. Code quality - DRY violations, error handling, tech debt
3. Tests - coverage, edge cases, failure scenarios
4. Performance - N+1 queries, memory, caching

For every issue it finds, Claude has to:
☑️ Describe the problem with file and line references 
☑️ Present 2-3 options (including "do nothing" when reasonable) 
☑️ Give effort, risk, impact, and maintenance cost for each 
☑️ Make an opinionated recommendation 
☑️ Ask for approval before moving forward

It even adapts to the size of the change:
• Big change → full review, up to 4 top issues per section
• Small change → one focused question per section

The real insight isn’t the prompt. It’s the workflow: review before implementation, tradeoffs before code, feedback before action.

For small teams without a senior engineer on every PR - you design the review process into your AI.
