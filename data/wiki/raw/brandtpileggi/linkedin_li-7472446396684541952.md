MCP vs A2A vs ACP vs ANP vs AG-UI vs AP2

To understand modern AI systems, you need to understand how these pieces fit together.

These protocols aren't competing for the same job, each one handles a different layer of how agents actually get work done:

1. 𝗠𝗼𝗱𝗲𝗹 𝗖𝗼𝗻𝘁𝗲𝘅𝘁 𝗣𝗿𝗼𝘁𝗼𝗰𝗼𝗹 (𝗠𝗖𝗣) 
↳ Connects an agent to outside tools and data through a standard client–server interface, so you skip custom code for every integration.

2. 𝗔𝗴𝗲𝗻𝘁𝟮𝗔𝗴𝗲𝗻𝘁 (𝗔𝟮𝗔) 
↳ Lets agents publish their capabilities, discover other agents, and exchange tasks, messages, and artifacts.

3. 𝗔𝗴𝗲𝗻𝘁 𝗖𝗼𝗺𝗺𝘂𝗻𝗶𝗰𝗮𝘁𝗶𝗼𝗻 𝗣𝗿𝗼𝘁𝗼𝗰𝗼𝗹 (𝗔𝗖𝗣) 
↳ Gives agents a lightweight, HTTP-native format for sending tasks, context, and responses through callable endpoints.

4. 𝗔𝗴𝗲𝗻𝘁 𝗡𝗲𝘁𝘄𝗼𝗿𝗸 𝗣𝗿𝗼𝘁𝗼𝗰𝗼𝗹 (𝗔𝗡𝗣)
↳ Lets agents find each other across an open network, agree on the rules, and exchange encrypted messages.

5. 𝗔𝗴𝗲𝗻𝘁 𝗨𝘀𝗲𝗿 𝗜𝗻𝘁𝗲𝗿𝗮𝗰𝘁𝗶𝗼𝗻 𝗣𝗿𝗼𝘁𝗼𝗰𝗼𝗹 (𝗔𝗚-𝗨𝗜)
↳ Standardizes communication between the frontend UI and the agent backend, streaming updates out to the user and sending events back in real time.

6. 𝗔𝗴𝗲𝗻𝘁 𝗣𝗮𝘆𝗺𝗲𝗻𝘁𝘀 𝗣𝗿𝗼𝘁𝗼𝗰𝗼𝗹 (𝗔𝗣𝟮) 
↳ Adds a standardized payment layer so agents can authorize, verify, and execute payments on behalf of users.

The key idea: these protocols don’t all solve the same problem.

Some connect agents to tools.
Some connect agents to other agents.
Some connect agents to users, networks, or payment systems.

Together, they turn isolated AI models into systems that can interact with the outside world.
