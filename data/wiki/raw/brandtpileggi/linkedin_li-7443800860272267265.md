Someone just open-sourced the "Docker for AI agents" 🤯

Define your agent once. Run it in Claude Code, OpenAI, CrewAI, LangChain, Google Gemini CLI, and OpenClaw.

Meet GitAgent. Your repo is the agent.

Right now every framework defines agents differently. Your CrewAI agent can't run in Claude Code. Your LangChain agent can't port to OpenAI.

You want to switch frameworks? You rewrite everything. Prompts, tools, memory, config. All of it. Again.

That's not a feature of the ecosystem. That's vendor lock-in wearing an open-source badge.

GitAgent replaces all of it with a standard folder structure.

agent.yaml for config. SOUL.md for personality. RULES.md for hard constraints. skills/ for capabilities. tools/ for integrations. memory/ for persistence.

Two files are required. Everything else is optional.

Then one command exports to any runtime.

Same repo runs in Claude Code, OpenAI Agents SDK, CrewAI, LangGraph, Google Gemini CLI, OpenClaw, and Cursor.

No rewrite. No adapter code. No framework-specific config.

Git becomes the supervision layer. Every prompt change is a commit. Every rollback is a checkout. Every agent review is a PR. Agent quality gets treated like code quality.

What makes this more than a packaging spec:
• Built-in segregation of duties for regulated industries. Define maker, checker, and auditor roles. 
• Sub-agents compose like submodules. Fork a public agent, swap its SOUL.md, plug in your own tools.
• CI/CD runs gitagent validate on every push. Bad agent configs don't ship.

Dockerfiles didn't replace container runtimes. They gave developers one format that every runtime could read.

GitAgent is that moment for agents.
