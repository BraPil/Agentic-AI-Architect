People think CLAUDE. md is just a notes file for Claude.

It's not.

It's the entire control system for how Claude Code thinks, acts, and stays safe inside your repo.

Here is the exact structure top engineers use:

☑ The Claude Code File Structure

claude/settings. json: committed permissions and hooks
claude/settings. local. json: gitignored, your personal overrides
claude/skills/code-review/SKILL. md: auto-triggers a code
claude/skills/testing/SKILL. md: auto-triggers your test
claude/skills/pr-description/SKILL. md: auto-writes PR
claude/skills/deploy/SKILL. md: auto-runs deploy checklist.
claude/agents/security-reviewer. md: a subagent that only reviews
claude/agents/test-writer. md: a subagent restricted to writing tests
claude/agents/docs-writer. md: a subagent restricted to writing docs
claude/agents/research. md: a subagent that reads and researches
claude/hooks. py: the script that enforces rules at compliance
claude/hooks/config/hooks-config. json: committed hook rules for repo
claude/hooks/scripts/block-sensitive-writes. sh: stop writing before start.
claude/commands/deploy. md: custom /deploy slash command.
claude/rules/architecture. md: the architecture rules claude never breaks.
gitignore: keeps your .env and secrets out of every commit.

That's why hooks are literally the only rule type that cannot fail.
