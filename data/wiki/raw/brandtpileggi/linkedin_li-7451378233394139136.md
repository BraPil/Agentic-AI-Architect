I'm starting a public experiment where I'm handing my entire codebase over to AI. No humans allowed to write or review a single line of code!

It's called a Dark Factory. And no, I wouldn't recommend it for reliable production code - yet. That's exactly why I'm calling it an experiment.

Dan Shapiro laid out five levels of AI coding autonomy earlier this year, using a self-driving car analogy. Level 3 is where reliable production AI coding lives right now: AI generates most of the code, but you're still reviewing every plan and every PR before it ships. That's what I teach in my Agentic Coding Course. You're in a self-driving car but still watching the road and ready to course correct if needed.

Level 5 is the Dark Factory - no steering wheel. Nothing gets reviewed. Even I'm a little uncomfortable with it! But this is where AI coding is heading, and nobody has run it as an open experiment before.

StrongDM did a version of this internally (32K lines shipped, zero human review, went viral in February), but you can't watch it work or participate. That's the gap I'm filling!

Here's how I'm building it, and this is where Archon (my open source harness builder) becomes the backbone.

Governance layer:

Three files loaded at the start of every workflow: MISSION.md (what the factory can build, what it must never touch), FACTORY_RULES.md (triage, PR size limits, quality gates, mandatory browser testing), CLAUDE.md (stack, layout, conventions). This is the only meaningful way I can stay aligned with a system that ships without asking me.

Archon workflows:

Archon is my open source harness builder - it turns your coding process into packaged up YAML workflows that can include both AI and deterministic steps. Three of them run the factory:

- Triage: classifies GitHub issues against MISSION.md + FACTORY_RULES.md
- Implement: runs in parallel for every accepted issue
- Validate: completely separate workflow to fully review PRs + merge

The validate workflow uses the "holdout" pattern from StrongDM. The validator only sees the user journey and the diff, zero context from development. No sycophancy, no rewriting tests to fake green.

If validation passes, it merges to main. No human approval at all.

The coding model is Minimax M2.7 routed into Claude Code via env vars. Purely an economics call, my Anthropic subscription would hit rate limits in hours at this throughput!

The app the factory is building is an AI Tutor, a RAG agent over my YouTube content. That part I'm genuinely excited about on its own.

Within a week or two it'll be deployed! You'll be able to file real GitHub issues and watch the factory accept, implement, validate, and ship them without me touching a line. Full architecture breakdown is on my YouTube channel!
